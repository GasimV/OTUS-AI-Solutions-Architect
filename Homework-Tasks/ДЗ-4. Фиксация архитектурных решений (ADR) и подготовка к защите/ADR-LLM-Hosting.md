# Title
ADR-001: Выбор хостинга LLM для Confluence Bot (SaaS vs Self-hosted)

# Status
Accepted

# Context
Проект — ИИ-бот для обработки клиентских запросов с интеграцией в ERP/БД и Confluence.  
Ограничения и требования:
- Деплой в VM заказчика, доступ по VPN/SSH, GitLab заказчика.
- Логи и персональные данные должны оставаться в периметре заказчика.
- Базовый RPS 5, пик 10; SLO по латентности ответа 10–16 секунд.
- Предпочтение open-source компонентам, облачные LLM ограничены токен‑бюджетом.
- Необходима готовность к масштабированию и будущей агентной архитектуре.

Рассматривались варианты:
1. Проприетарная облачная LLM (SaaS, например GPT-5).
2. Open-source LLM на своих серверах (Self-hosted, например Llama 3).

# Decision
Выбираем self-hosted open-source LLM (Llama 3) в периметре заказчика с управляемым inference-сервисом, очередями и прокси для горизонтального масштабирования.

# Consequences

Pros
- Данные и логи остаются в периметре заказчика, проще соответствовать требованиям ИБ.
- Нет зависимости от внешних лимитов и политик провайдера; проще контролировать RPS и доступность.
- Стоимость прогнозируемая и под контролем заказчика (CAPEX/OPEX на инфраструктуру).
- Архитектура лучше согласуется с on-prem деплоем и будущей агентной шиной событий.

Cons
- Ниже качество ответов на сложных кейсах по сравнению с топ‑SaaS моделями.
- Выше сложность поддержки: обновления, мониторинг, оптимизация инференса.
- Требуются ресурсы GPU и компетенции в эксплуатации моделей.
- Риски по латентности при высокой нагрузке, если не настроено масштабирование и кэширование.
