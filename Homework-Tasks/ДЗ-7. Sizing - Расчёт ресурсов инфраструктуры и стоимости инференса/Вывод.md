# Sizing & TCO: Llama-3-70B — Краткий вывод

## 1. Hardware Sizing

### Расчёт VRAM

| Метрика | FP16 | INT4 |
|---------|------|------|
| Вес модели | 70B * 2 байта = **130.39 GiB** | 70B * 0.5 байта = **32.60 GiB** |
| KV Cache (33 concurrent, 2K tokens, FP16 dtype) | ~2.50 GiB | ~2.50 GiB |
| **Итого VRAM** | **~132.89 GiB** | **~35.10 GiB** |

### Формула VRAM
```
Model_GiB  = параметры * 10^9 * байт_на_параметр / 1024^3

KV_Cache_MiB (на 1 запрос) = 2 * layers * num_kv_heads * head_dim * seq_len * dtype_bytes / 1024^2
KV_Cache_GiB (итого)       = KV_Cache_MiB * concurrency / 1024

VRAM_GiB = Model_GiB + KV_Cache_GiB
```

### Параметры Llama-3-70B
- layers = 80, head_dim = 128, num_kv_heads = 8 (GQA), seq_len = 2048
- KV cache dtype: FP16 (2 байта) — переключаемый флаг (можно задать INT4 = 0.5 байта)

### Требуемые GPU (для 1000 RPM)

| Конфигурация | GPU на реплику | Реплик | GPU итого |
|-------------|---------------|--------|-----------|
| FP16, A100-80 GiB | 2 | 6 | **12** |
| **INT4, A100-80 GiB** | **1** | **4** | **4** |
| INT4, L4-24 GiB | 2 | 5 | 10 |

**Расчёт concurrency (закон Литтла):**
- RPM = 1000 -> RPS = 16.67
- Средняя латентность = 2 сек
- Concurrent requests = 16.67 * 2 = **~33 запроса одновременно**
- A100 с continuous batching обрабатывает ~8-10 concurrent запросов -> нужно 4 реплики (INT4)

## 3. Оптимизация

### Эффект оптимизаций (INT4, A100-80 GiB, Cloud.ru)

| Сценарий | GPU | ₽/мес | Экономия |
|----------|-----|-------|----------|
| Baseline (без оптимизации) | 4 | ~1 430 800 | — |
| + Continuous Batching | 3 | ~1 073 100 | 25% |
| + vLLM (PagedAttention) | 2 | ~715 400 | **50%** |
| **+ vLLM + Batching + INT4** | **2** | **~715 400** | **50%** |

**Как это работает:**
- **Continuous Batching** — объединяет запросы в динамические батчи, повышая утилизацию GPU с ~30% до 80%+
- **vLLM (PagedAttention)** — управляет KV Cache как виртуальной памятью, устраняя фрагментацию и позволяя обслуживать больше concurrent запросов на одном GPU
- **Квантование INT4** — сокращает VRAM в 4x при потере качества <1% (AWQ/GPTQ)

## 4. Рекомендация

### Оптимальная конфигурация:

| Параметр | Значение |
|----------|----------|
| Модель | Llama-3-70B **INT4** (AWQ квантование) |
| GPU | **2x NVIDIA A100 80 GiB** |
| Inference Engine | **vLLM** с PagedAttention + Continuous Batching |
| Облако | **Cloud.ru** (дешевле на ~9%) |
| Стоимость | **~715 400 ₽/мес** |

### Почему именно эта конфигурация:

1. **INT4 квантование** снижает VRAM со 130.39 GiB до 32.60 GiB — модель помещается на 1 GPU
2. **vLLM** увеличивает throughput в 3-5x через PagedAttention, позволяя обслуживать 1000 RPM на 2 GPU
3. **Cloud.ru** дешевле Yandex Cloud на ~9% для A100 инстансов
4. **Итоговая экономия** vs FP16 без оптимизации: **~85%** (с ~4.3M ₽ до ~715K ₽/мес)
5. **Потеря качества** при INT4 (AWQ): менее 1% на стандартных бенчмарках — приемлемо для production
